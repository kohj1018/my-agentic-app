## 출력 원칙
- 사실, 가정, 열린 질문을 구분한다.
- 스택이 명확하지 않으면 stack-specific 자동화는 만들지 않는다.
- 상위 문서와 하위 문서의 역할을 섞지 않는다.
- 너무 많은 문서를 한 번에 만들지 않는다.
- 마지막에는 갱신한 파일 목록과 남은 미결정 사항을 요약한다.
