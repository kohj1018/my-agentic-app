# 목적
`docs/10-charter/PROJECT_CHARTER.md`를 생성하거나 갱신한다.

## 작업 절차
1. 기존 `docs/10-charter/PROJECT_CHARTER.md`를 읽는다.
2. 프로젝트 요약, 대상 사용자, 문제, 목표, 비목표, 성공 기준, 제약 조건을 정리한다.
3. 확실하지 않은 내용은 가정으로 표시한다.
4. 추측으로 메우지 말고 열린 질문을 남긴다.
5. 문서는 짧고 구체적으로 유지한다.

## 출력 원칙
- 흩어진 별도 문서를 만들지 않는다.
- 구현 체크리스트를 이 문서에 넣지 않는다.
- 사실, 가정, 열린 질문을 구분한다.
