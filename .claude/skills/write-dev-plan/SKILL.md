# 목적
`docs/DEV_PLAN.md`를 생성하거나 갱신한다.

## 작업 절차
1. `docs/PRD.md`를 먼저 읽는다.
2. 아키텍처, 데이터 모델, API 방향, 페이지/컴포넌트 구조, 마일스톤을 도출한다.
3. 구현 중심으로 정리한다.
4. 리스크와 미확정 사항을 명시한다.
5. 과도한 설계는 피한다.

## 출력 원칙
- 결과는 `docs/DEV_PLAN.md`에 반영한다.
- 한 번에 거대한 구현보다 단계별 마일스톤으로 나눈다.
- 이후 구현 에이전트가 바로 실행 가능한 수준으로 잘게 쪼갠다.
