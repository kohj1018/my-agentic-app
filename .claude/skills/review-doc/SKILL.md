# 목적
대상 문서를 리뷰하고 바로 반영 가능한 피드백을 만든다.

## 작업 절차
1. 대상 문서를 읽는다.
2. 누락, 모순, 애매함, 엣지 케이스, 검증 불가능한 요구사항을 찾는다.
3. 결과를 P0, P1, P2로 나눈다.
4. 어떤 섹션을 어떻게 수정하면 좋을지 제안한다.

## 출력 원칙
- 막연한 칭찬은 피한다.
- 상위 설계 문제와 하위 구현 문제를 구분한다.
- 바로 수정 가능한 지적을 우선한다.
