# 목적
대상 문서를 검토하고 바로 수정 가능한 피드백을 만든다.

## 작업 절차
1. 대상 문서를 읽는다.
2. 모호함, 모순, 숨은 복잡도, 엣지 케이스 누락, 검증 불가능한 요구사항을 찾는다.
3. 문제를 P0, P1, P2로 분류한다.
4. 가능하면 어떤 섹션을 어떻게 고치면 좋을지 제안한다.

## 출력 원칙
- 막연한 칭찬은 피한다.
- 즉시 반영 가능한 구체적 지적을 우선한다.
- 수정 필요성이 큰 항목부터 제시한다.
