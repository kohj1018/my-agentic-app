# 목적
`docs/30-workitems` 아래 문서를 생성하거나 갱신한다.

## 작업 절차
1. 관련 charter와 architecture 문서를 읽는다.
2. 작업 단위를 milestone, feature, task 중 적절한 레벨로 나눈다.
3. 각 문서의 범위와 비범위를 명확히 적는다.
4. 관련 문서 링크를 함께 기록한다.
5. 검증 포인트와 완료 기준을 포함한다.

## 출력 원칙
- 서로 다른 추상화 레벨을 한 문서에 섞지 않는다.
- milestone은 큰 목표, feature는 사용자 가치, task는 구현 단위로 유지한다.
- 하위 문서는 상위 문서를 링크한다.
