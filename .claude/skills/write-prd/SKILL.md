# 목적
`docs/PRD.md`를 생성하거나 갱신한다.

## 작업 절차
1. 기존 `docs/PRD.md`가 있으면 먼저 읽는다.
2. 서비스 요약, 타깃 사용자, 문제, 목표, 비목표, 핵심 사용자 흐름을 정리한다.
3. 확실하지 않은 내용은 가정으로 표시한다.
4. 추측으로 메우지 말고 열린 질문을 남긴다.
5. 문서는 짧고 구체적으로 유지한다.

## 출력 원칙
- 새 문서를 흩뿌리기보다 기존 파일을 갱신한다.
- 검증 가능한 요구사항을 우선한다.
- 사실, 가정, 열린 질문을 구분한다.
